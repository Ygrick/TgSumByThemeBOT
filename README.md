# TgSumByThemeBOT

Telegram-бот на Python для групповых чатов с локальной SQLite БД.

## Возможности

1. Оповещение всех известных участников группы: `/notify_all`.
2. Анализ обсуждений за последние 10 часов с выделением топ-3 тем: `/analyze_topics`.
3. Выдача краткой сводки по теме из БД: `/topic <id>`.
4. Просмотр последних 10 сохраненных тем: `/latest_topics`.

## Технологии

- Python
- [python-telegram-bot](https://python-telegram-bot.org/)
- OpenAI Python Client с `base_url` на OpenRouter
- OpenRouter (модель по умолчанию: `openai/gpt-oss-120b`)
- uv (управление зависимостями)
- SQLite (локальная БД без отдельного сервера)

## Быстрый запуск

1. Установите зависимости:

```bash
uv sync --locked
```

2. Создайте `.env`:

```bash
cp .env.example .env
```

3. Заполните обязательные переменные:

- `TELEGRAM_BOT_TOKEN`
- `OPENROUTER_API_KEY`

4. Проверьте base URL:

- корректно: `OPENAI_BASE_URL=https://openrouter.ai/api/v1`
- некорректно: `.../api/v1/chat/completions`

5. Запустите бота:

```bash
uv run python src/main.py
```

## Подробно По Командам

`/help`
- Что делает: показывает подробную справку.
- Когда использовать: если нужно быстро понять, какая команда нужна под задачу.

`/notify_all [текст]`
- Что делает: отправляет текст и упоминает всех известных участников из локальной БД.
- Если текст не указан, бот отправляет только упоминания участников.
- Когда использовать: срочное уведомление, релиз, сбор на звонок, эскалация.
- Примечание: не использует LLM, работает только на данных БД.

`/analyze_topics`
- Что делает: анализирует сообщения за последние 10 часов, сохраняет до 3 тем и сводки в БД.
- LLM-запросы: 1 запрос на весь анализ (темы + краткие сводки сразу вместе).
- Когда использовать: понять, что обсуждалось в последнее время без чтения всего чата.
- Что получаете: лаконичный список тем с `id` + кнопки выбора темы (без лишнего служебного текста).

`/topic <id>`
- Что делает: возвращает краткую сводку темы из БД по `id`.
- LLM-запросы: 0 (сводка читается из БД, повторный запрос к модели не нужен).
- Когда использовать: после `/analyze_topics` или `/latest_topics`.

`/latest_topics`
- Что делает: показывает последние 10 тем из БД.
- Когда использовать: чтобы посмотреть историю уже найденных тем.
- Что получаете: список тем, где дата в формате `19 февраля` кликабельна и ведет на первое сообщение темы, плюс кнопки для мгновенного открытия сводки.

## Эффективность И Язык Ответов

- `/analyze_topics` делает один LLM-запрос и получает сразу список тем со сводками.
- `/topic <id>` не обращается к LLM, потому что сводка уже сохранена в БД.
- Промпты принудительно настроены на русский язык для тем, сводок и открытых вопросов.
- В LLM-контекст передаются только релевантные сообщения окна анализа: без служебных команд, без дублей подряд и без лишней мета-информации.

## Примеры Использования

Пример 1. Ежедневный обзор обсуждений
1. Выполнить `/analyze_topics`.
2. Выбрать нужную тему кнопкой или выполнить `/topic <id>`.

Результат: за 1-2 минуты понятно, о чем говорили и к чему пришли.

Пример 2. Срочное групповое уведомление
1. Выполнить `/notify_all`.
2. Или отправить уведомление с текстом: `/notify_all Релиз переносим на 30 минут, проверьте статус задач`.

Результат: бот отправит единое сообщение с упоминаниями участников.

## Команды В Интерфейсе Telegram

Сделано в коде: бот при старте регистрирует команды через Telegram Bot API (`setMyCommands`), поэтому они видны в меню команд в клиенте Telegram.

Как это обычно выглядит:
- в поле ввода нажимаете `/`
- Telegram показывает список доступных команд с описанием
- можно выбрать команду тапом, не печатая вручную

Если список не появился сразу:
1. Перезапустите бота.
2. Перезайдите в чат с ботом.
3. Введите `/help` вручную один раз.

## Почему Могла Быть Ошибка 404 В Аналитике

Симптом:
- запрос идет на `.../chat/completions/chat/completions`

Причина:
- в `OPENAI_BASE_URL` (или `OPENROUTER_BASE_URL`) указан полный endpoint `.../chat/completions`, а OpenAI client добавляет этот путь сам.

Исправление:
- использовать `OPENAI_BASE_URL=https://openrouter.ai/api/v1`
- в проекте уже добавлена нормализация URL, чтобы автоматически обрезать лишний хвост `/chat/completions`

## Структура БД

Используется один файл SQLite (`bot.db` по умолчанию):

- `participants` - участники, никнеймы, last_seen.
- `messages` - история сообщений.
- `topics` - найденные темы и их сводки.
- `topic_messages` - связь тем и сообщений.
- `open_question_reports` - результаты анализа открытых вопросов.

## Важные Замечания

- Для сбора истории сообщений бот должен видеть сообщения в группе.
- Обычно для этого нужно отключить Privacy Mode у бота в BotFather.
